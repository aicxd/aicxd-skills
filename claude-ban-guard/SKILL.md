# claude-ban-guard —— Claude 封号自检与防封

## 这个 skill 干嘛的

Anthropic 在 Claude Code 里(自 2.1.91 / 2026-04-03 起)埋了一套**隐写术标记**:本地判断你是不是中国用户,
是的话就在发给服务端的系统提示词 `Today's date is …` 这行里,把单引号和日期分隔符换成肉眼看不出、但机器能解码的
Unicode 变体,服务端据此对中国大陆请求做风控(防倒卖/中转/蒸馏),这也是国区账号被大面积封的抓手之一。

**隐写标记这层不靠 IP 采集**，只看 3 个本地信号（即使挂着海外代理，本地信号没清干净照样被打标记）:

1. **系统时区** —— Node/Claude 读到的 IANA 时区是不是 `Asia/Shanghai` / `Asia/Urumqi`。
2. **`ANTHROPIC_BASE_URL`** —— 官方用户根本不设它;设了非官方地址就是信号。
3. **中转站域名** —— 上面那个地址的域名会被拿去和内置的 147 个中转站/大厂/11 个 AI 实验室关键词列表比对
   (列表是 base64 + XOR(key=91) 混淆的)。

本 skill 只做两件事:**① 只读扫出你命中哪些信号并判定风险;② 给一步步修复步骤,安全的可代改(先问),红线的只给指引。**

## 铁律(必须遵守)

- **扫描永远只读**,不碰任何文件、不设任何变量。
- 触碰**环境变量 / `.env` / 系统时区 / `ANTHROPIC_BASE_URL` / 系统配置**属于用户红线:**先说清「改什么+改了会怎样+风险」再问,拿到明确同意才动。绝不自动改。**
- 诚实:这些手段只能**降低被标记概率**,不能保证不封;且中国大陆本就不在 Anthropic 支持区,用中转已违反 ToS。不打包票。

## 第 1 步:跑只读自检

在 PowerShell 里:

```
pwsh -NoProfile -File "<此 skill 目录>/scan.ps1" -ProjectDir "<当前项目根,默认 cwd>"
```

脚本会逐项打印,并给每个信号 GREEN / YELLOW / RED 判定 + 一个总体结论。把输出**原样念给用户看关键行**,再按下面解读。

`api.anthropic.com` 的地址 → 不通过;否则通过。**任一项不通过 = 请求正被打上「中国用户」标记。**

## 第 3 步:按信号给修复步骤

对每个 RED 信号,给用户下面对应的步骤。**动手前先按铁律讲清影响并征得同意。**

### 信号1 RED —— 系统时区是 Asia/Shanghai / Urumqi

关键点:**不用改 Windows 系统时区**(你日常还得看北京时间)。Claude Code 走 Node,而 Node 认 `TZ` 环境变量,
只需在**启动 Claude 的那个环境**里把 `TZ` 设成非中国时区(如 `America/Los_Angeles`、`Asia/Tokyo`、`UTC`),
Node/Claude 就会读到这个值,而系统时钟显示不变。

- **对话用 Claude Code**:在启动它的终端 / 快捷方式里先 `set TZ=America/Los_Angeles`(cmd)或 `$env:TZ="America/Los_Angeles"`(PowerShell)再起 `claude`。要长期生效可加到用户级环境变量——**这条属红线,先问**。
- **无头调用 / 自动化脚本**:在对应的 `.bat` / 包装脚本开头加一行 `set TZ=America/Los_Angeles`。改的是项目脚本文件,**先说清后再改**。
- 验证:重新跑 `scan.ps1`,信号1 应变 GREEN;或 `node -e "console.log(Intl.DateTimeFormat().resolvedOptions().timeZone)"` 确认。

> 注意:改 `TZ` 会影响该进程里所有按本地时区算时间的逻辑(日志时间戳、按天取数的窗口等)。改前务必确认时区敏感逻辑用的是显式时区而非机器本地时区,否则会串日期。

### 信号2 RED —— 设了非官方 ANTHROPIC_BASE_URL(最强信号,属红线)

这是最强的中国标记来源,但也最敏感(改它 = 改你怎么连 Claude),**只给指引,让用户决定,绝不自动改**:

1. **能走官方就走官方**:有合规海外网络 + 官方账号/订阅时,**删掉** `ANTHROPIC_BASE_URL`(环境变量 / settings.json 的 `env` 块 / `.env` 三处都要清),直连 `api.anthropic.com`,这条信号直接消失。脚本已列出它在哪几处被设。
2. **必须用中转/自建代理**:确认代理域名不在那 147 个列表里(用独立自建域名),但列表会更新,不是长久之计;公司内网代理域名同样会被命中。
3. 用**官方 CLI 订阅登录**(不设 key、不设 base_url)跑 `claude`,是天然规避——尽量维持。

### 信号3 —— 域名命中列表

和信号2 是同一处(BASE_URL 的域名)。脚本会把非官方地址的域名单独列出来,让用户眼看是不是阿里/字节/中转站之类的大名。处置同信号2。

### 信号5 —— 浏览器加固(浏览器登录 / 管理 Claude 账号时)

只在你**用浏览器登录 claude.ai / 管理账号**时相关;纯命令行 `claude` 不受影响。目标:让浏览器每一项都跟你的出口 IP 国家一致,别在登录时露出中国。脚本能 best-effort 读到浏览器语言和加固启动参数,读不到的(WebRTC 实测/定位/时区)得你自己在浏览器里查。**改启动参数/系统 IPv6 属红线,先讲清影响再动。**

**Chrome 启动参数(真·旗标,加在快捷方式「目标」末尾或启动命令后):**
```
chrome.exe --lang=en-US --force-webrtc-ip-handling-policy=disable_non_proxied_udp
```
- `--lang=en-US` —— 界面语言设成跟 IP 一致(美国 IP 就 en-US)。
- `--force-webrtc-ip-handling-policy=disable_non_proxied_udp` —— 强制 WebRTC 只走代理 UDP,不暴露隧道外真实 IP(等价于装 WebRTC Control 扩展)。

**要在设置/系统里手动做的(没有干净的启动参数):**
1. **语言一致** —— `chrome://settings/languages` 把「English (United States)」拖到最上、删掉中文,让 `Accept-Language` 头也不露中文。
2. **堵 WebRTC** —— 用上面的启动参数;或装扩展 WebRTC Control。测:browserleaks.com/webrtc,看 Public IP 别露中国。
3. **关定位** —— `chrome://settings/content/location` 设为「不允许网站获取您的位置」;再关 Windows 定位(设置→隐私和安全性→定位)。
4. **时区对齐 IP** —— 网站读的是**操作系统时区**(Chrome 无独立时区设置,跟随系统)。系统时区若已跟出口 IP 一致(如美国 IP + America/Los_Angeles)就无需再动;若系统是中国时区又不想改系统显示,用时区伪装扩展或 DevTools `Emulation.setTimezoneOverride`。
5. **关 IPv6** —— 这是网卡/系统层不是 Chrome:网络适配器属性里取消勾选「Internet 协议版本 6 (TCP/IPv6)」,或只用纯 IPv4 出口的代理节点,避免 IPv6 绕过代理直连暴露。

> 更稳的做法:登录/管理 Claude 账号用一个**专用浏览器 profile**(甚至专用反检测浏览器),把上面几项一次配好,别和日常中文浏览混用。

### (进阶,可选)清洗隐写标记本身

标记是 Claude Code 在本地拼进 prompt 后才发出的,理论上可在出站前用一个本地代理把 `Today's date` 那行的
撇规范化回 `U+0027`、斜杠改回 `-`。但这要自建 MITM 代理做字符规范化,脆弱且 Anthropic 随时能换载体,**收益不稳、成本高,一般不推荐**,只在用户明确要对抗时再展开。

## 第 4 步:账号 / 行为韧性(脚本查不到,口头提醒)

标记只是识别手段,**真正封号还看行为**。即使全 GREEN,也提醒用户:

- 别用单个订阅账号跑超大量自动化/蒸馏式调用;别多账号共用一个出口 IP。
- 注册地、支付方式、网络出口、时区尽量一致且非中国,减少风控画像异常。
- **B 计划最重要**:关键链路别单点依赖 Claude 账号。脚本会报 `DEEPSEEK_API_KEY 已配置` —— 在调用链路上做「Claude 任意失败即落 DeepSeek」的兜底是对冲封号的正解。若没配 DeepSeek,提醒补上(改 `.env` 属红线,✋ 手动)。

## 收尾

- 严格按第 2 步的模板输出报告,不要另起格式。
- 报告末尾把待办按「🤖 Claude 可代做(需确认)/ ✋ 需你手动」两类列清,让用户一眼知道下一步谁动手。
