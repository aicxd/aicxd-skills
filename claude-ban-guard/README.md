# Claude 封号:识别逻辑与处理

> 一句话:Anthropic 在 Claude Code 里埋了**隐写术标记**,靠 3 个本地信号(系统时区 / `ANTHROPIC_BASE_URL` / 中转站域名)判断你是不是中国大陆用户,是的话就在发往服务端的系统提示词里做肉眼不可见的标记,服务端据此风控、封号。该逻辑**自 Claude Code 2.1.91(2026-04-03)起就在包里**。

---

## 一、封号识别逻辑

**关键:它不看 IP**——传统地域封锁靠 IP,挂代理就能绕;这套走的是 3 个跟网络出口无关的本地信号。

### 信号 1 —— 系统时区
Claude Code 基于 Node,读本地 **IANA 时区**。若为 `Asia/Shanghai`(上海)或 `Asia/Urumqi`(乌鲁木齐),即判为中国大陆。挂代理改的是网络出口,但时区要看时间几乎没人改,所以最稳。

### 信号 2 —— `ANTHROPIC_BASE_URL`
官方用户用 API **根本不设**这个变量,直连 `api.anthropic.com`。国内只能把它指向中转站,大厂则自建内网代理。**只要设了非官方地址,本身就是信号。**

### 信号 3 —— 中转站 / 大厂 / 实验室域名
拿信号 2 的域名和一份内置列表比对:**147 个**已知中转站 / 国内大厂内网(美团、网易、百度、携程、小红书、阿里、蚂蚁、字节、京东、B站……)+ **11 个**中国 AI 实验室关键词(DeepSeek、Kimi/Moonshot、智谱、MiniMax、百川、阶跃……)。列表非明文:先 base64、再 XOR 异或(**密钥 = 91**)混淆。

### 识别出来后怎么打标记(隐写术 steganography)
每次请求前系统提示词会拼一行 `Today's date is 2026-06-30`。命中信号后对这行做**两处肉眼不可见**的改动:

1. **`Today's` 的单引号**:正常是 `U+0027`。命中后换成三个"长得一样"的 Unicode 之一——`U+2019`(右单引号)、`U+02BC`(修饰字母撇)、`U+02B9`(音标 prime)。
2. **日期分隔符**:时区是上海/乌鲁木齐时,把 `-` 换成 `/`,变 `2026/06/30`。

**两处各对应一个信号**(判断顺序):先看**时区**→是中国大陆就把日期改斜杠;再看 **API 地址**→指向官方就什么都不做,指向非官方才继续比对域名/实验室关键词,命中不同类别就把**那个撇**换成不同 Unicode。即:**斜杠 = 时区信号,撇的变体 = BASE_URL/域名信号**。两者叠加 = **2~3 bit 分类标记**,服务端解码这两处即知是否来自中国大陆及命中类别——你在任何编辑器/终端里看那行字都跟正常版一模一样。

### 旁路:封号邮件里的地址追踪器
除 Claude Code 本体,Anthropic 在**封号邮件里也埋了地址追踪器**(邮件像素/追踪链接):一打开就回传"何人、何地、何时打开"。这条本地扫描查不到,应对见下。

### 第二条封号面:网络环境一致性(账号风控)
上面的隐写标记**不看 IP**;但 Claude **账号级风控**是另一条线,会综合**出口 IP 国家 / DNS / WebRTC / IPv6 / 系统时区 / 系统语言**判断账号是否可疑。挂代理时最容易翻车的不是"没伪装",而是**伪装不一致**——出口 IP 是美国,但 DNS 泄露到国内、WebRTC 暴露真实 IP、IPv6 直连绕过代理、系统语言还是 zh-CN……这些互相矛盾的信号反而更像"在藏什么"。核心叫**环境一致性**:IP 国家 / DNS / 时区 / 语言全部指向同一个非中国地区,且无 WebRTC / IPv6 泄露。

## 二、怎么处理

> 铁律:动**环境变量 / `.env` / 系统时区 / `ANTHROPIC_BASE_URL` / 系统配置**都是红线,改前先讲清影响再动。下面标了"谁处理"。

### 解信号 1(时区)—— 🤖 可代做(需确认)
**不改 Windows 系统时区**(日常还得看北京时间)。Node 认 `TZ` 环境变量,只在**启动 Claude 的那个环境**里把 `TZ` 设成非中国时区即可,系统时钟显示不变:

- **对话用**:启动终端里先设,再起 `claude`
  - PowerShell:`$env:TZ="America/Los_Angeles"`
  - cmd:`set TZ=America/Los_Angeles`
- **无头调用 / 自动化脚本**:脚本开头加一行 `set TZ=America/Los_Angeles`
- **长期生效**:加到用户级环境变量(属红线,先问)
- **验证**:`node -e "console.log(Intl.DateTimeFormat().resolvedOptions().timeZone)"` 应显示非中国时区
- ⚠️ 副作用:改 `TZ` 影响该进程里所有按本地时区算时间的逻辑(日志时间戳、按天取数窗口)。改前确认时区敏感逻辑用的是**显式时区**而非机器本地时区。

### 解信号 2 / 3(BASE_URL / 域名)—— ✋ 手动(最敏感,只给指引)
1. **能走官方就走官方**:有合规海外网络 + 官方账号/订阅时,把 `ANTHROPIC_BASE_URL` 从**三处都清掉**(环境变量 / `settings.json` 的 `env` 块 / `.env`),直连 `api.anthropic.com`,信号直接消失。
2. **必须用中转**:换独立自建域名(别用已知中转站/大厂域名),但列表会更新,不是长久之计;公司内网代理域名同样会被命中。
3. **最省心**:用官方 CLI 订阅登录(不设 key、不设 base_url)跑 `claude`——天然规避。

### 解封号邮件追踪器 —— ✋ 手动
邮件客户端关闭"远程图片/外链自动加载",别直接点封号邮件里的链接。设置一次即可。

### 账号 / 行为韧性(标记只是识别,封号还看行为)
- 别用单账号跑超大量自动化/蒸馏式调用;别多账号共用一个出口 IP。
- 注册地、支付、网络出口、时区尽量一致且非中国。
- **B 计划**:关键链路别单点依赖 Claude。在调用链路上做 Claude 失败即落 DeepSeek 的兜底是正解;若没配 `DEEPSEEK_API_KEY` 就补上(改 .env 属红线,✋ 手动)。

### 解网络环境一致性 —— ✋ 手动(挂代理相关)
用浏览器逐项自查并对齐:IP `ipinfo.io` → DNS 泄露 `dnsleaktest.com` → WebRTC `browserleaks.com/webrtc` → IPv6 `test-ipv6.com`。
- **关 IPv6**:Windows 适配器属性里取消勾选 IPv6,避免 IPv4 走代理、IPv6 直连泄露。
- **防 DNS 泄露**:Clash/Verge/Mihomo 开 `dns: { enable: true, enhanced-mode: fake-ip }`(DNS 显示 `198.18.x` = fake-ip 已生效,代理接管解析)。
- **关 WebRTC(必测,最易暴露真实国籍)**:先到 `browserleaks.com/webrtc` 看 Public IP 是否露出中国 IP;露了就关——Chrome/Edge 装扩展 **WebRTC Control**;Firefox `about:config` 设 `media.peerconnection.enabled=false`;关后重测直到不再露中国 IP。**仅命令行用 Claude 不受影响,浏览器登录/管理账号才受影响**。
- **统一时区 + 语言**:出口 IP 是哪国,系统/浏览器时区语言就对齐哪国。
- **换干净节点**:还不一致就换原生 IP(Residential/ISP)、非中转机场节点。

### 用 `claude-ban-guard` skill 一键判断处理

支持 SKILL.md 的 Agent 平台(Claude Code / Codex / OpenCode / OpenClaw)均可使用,全局 skill 任何项目都能触发。

- **触发**:说「封号自检」「会不会被封」「查下时区/BASE_URL」,或 `/claude-ban-guard`。
- **它做什么**:跑只读扫描 `scan.ps1`(查时区 / `ANTHROPIC_BASE_URL`(三处)/ 中转站域名 / 出口 IP 国家+环境一致性 / 版本 / DeepSeek 兜底)→ 按固定模板输出《Claude / Codex 账号安全自查报告》逐项 ✅/⚠️/❌ → 有不通过项按处理归属表给方案并标清谁动手。
- **手动跑脚本**:
  ```
  pwsh -NoProfile -File "<skill 目录>/scan.ps1" -ProjectDir "<项目根>"
  ```
- **何时复检**:配置漂移会重新命中——**Claude Code 升级后、改连接方式后、换机器后**都重跑。
